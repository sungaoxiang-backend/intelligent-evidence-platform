# 电子数据真实性审查

## 快速开始

当用户问"微信证据怎么确认"或"电子数据真实性怎么判断"时：

### Step 1: 审查八大因素（《证据规定》第93条）

按以下因素逐项审查：

| 审查因素 | 检查要点 |
|----------|----------|
| **硬件软件环境** | 计算机系统是否完整、可靠？ |
| **运行状态** | 系统是否正常运行？异常时有无影响？ |
| **监测手段** | 有无有效的防错监测手段？ |
| **完整保存** | 是否被完整保存、传输、提取？ |
| **正常活动** | 是否在正常往来活动中形成存储？ |
| **保存主体** | 保存、提取主体是否适当？ |
| **生成依赖** | 生成、存储、传输依赖的环境是否可靠？ |
| **其他因素** | 影响完整性和可靠性的其他因素 |

### Step 2: 判断完整性

**关键问题**：
- 是否被剪辑、篡改？
- 是否选择性提供？
- 是否保留了原始载体？

**警惕信号**：
- 只有截屏，没有原始记录
- 关键内容缺失
- 时间戳不连续

### Step 3: 推定真实的情形（《证据规定》第94条）

以下情形，法院可直接确认真实性：

| 推定情形 | 说明 |
|----------|------|
| **于己不利** | 当事人提交的对自己不利的电子数据 |
| **第三方平台** | 中立第三方平台提供或确认的数据 |
| **正常业务** | 在正常业务活动中形成的数据 |
| **档案管理** | 以档案管理方式保管的数据 |
| **约定方式** | 当事人约定方式保存、传输、提取的数据 |
| **公证确认** | 经公证机关公证的电子数据 |

### Step 4: 应对质疑

当对方质疑电子数据真实性时：

**可反驳的情形**：
- 硬件软件环境不完整、不可靠
- 系统曾处异常状态
- 保存、提取方法不可靠
- 不是正常业务活动中形成
- 保存主体不当

**应对策略**：
1. 补充第三方存证或公证
2. 提供系统正常运行的证据
3. 提交完整记录反驳选择性提供指控

## 常见电子数据类型

### 微信/QQ记录
- 需原始载体
- 核对微信号实名信息
- 展示完整对话记录

### 电子邮件
- 显示完整邮件头信息
- 保留原始邮件
- 导出时带时间戳

### 网页/博客
- 保存完整网页快照
- 记录URL和保存时间
- 建议公证保全

### 电子合同
- 保留签署时的完整系统环境
- 保存电子签名验证信息
- 提供第三方平台证明

## 证明力比较

| 证据形式 | 证明力 | 建议 |
|----------|--------|------|
| 公证电子数据 | 很高 | 重要证据首选 |
| 第三方存证 | 高 | 一般证据推荐 |
| 原始载体完整 | 中高 | 需核验 |
| 仅截屏/复印件 | 低 | 需其他证据佐证 |
| 选择性提供 | 很低 | 可能不被采信 |
