# 技能类型识别指南

本文档帮助 Agent 分析文章内容，判断应生成何种类型的技能。

> [!IMPORTANT]
> 类型判定是**自动**的，基于文章内容特征，不是用户选项。

---

## 快速识别决策树

```
文章内容是什么？
│
├─ 主要是【定义/概念/法条/规则/标准】
│   └─→ 知识型
│
├─ 主要是【步骤/流程/操作/SOP/检查清单】
│   └─→ 能力型
│
└─ 两者都有，且：
    ├─ 可以分离 → 分别处理，可能生成两个独立 skill
    └─ 强耦合不可分离 → 混合型（单一 SKILL.md 内分层组织）
```

---

## 知识型特征

**核心本质**：静态语义层 - 定义"是什么"

**典型内容**：
- 法律条文及司法解释
- 术语定义和概念说明
- 标准规范和要求
- 分类体系和类型划分
- 判断规则和适用条件

**识别关键词**：
- "是指"、"定义为"、"包括"
- "第X条"、"规定"、"标准"
- "类型"、"分为"、"属于"

**SKILL.md 组织方式**：
- 概念速查表
- 规则索引
- 分类对照表
- 全部引用 references/

---

## 能力型特征

**核心本质**：主动计算层 - 定义"怎么做"

**典型内容**：
- 操作步骤和流程
- 审查要点和检查清单
- 决策分支和条件判断
- 输入输出规范
- 常见问题处理方法

**识别关键词**：
- "步骤"、"流程"、"方法"
- "首先"、"然后"、"最后"
- "如果...则"、"当...时"
- "检查"、"验证"、"确认"

**SKILL.md 组织方式**：
- Step 1/2/3... 结构
- 决策表格
- 检查清单
- 引用 references/ 中的原文依据

---

## 混合型特征

**核心本质**：知识层为能力层提供语义支持

**判定条件**（必须同时满足）：
1. 文章中**既有**概念定义/规则，**又有**操作流程
2. 操作流程**依赖**概念定义才能执行
3. 分离两者会导致技能**不完整**

**SKILL.md 组织方式**：
```markdown
# 主题

## 第一部分：知识基础
[概念定义、规则索引]

## 第二部分：操作流程
[SOP，步骤中引用第一部分的概念]
```

---

## 常见判断示例

| 原文类型 | 判定 | 理由 |
|---------|------|-----|
| 《民事诉讼法》第66条 | 知识型 | 纯法条定义 |
| 微信证据审查操作手册 | 能力型 | 纯操作步骤 |
| 微信证据规则+审查流程 | 混合型 | 流程依赖规则判断 |
| 电子数据类型分类表 | 知识型 | 分类定义 |
| PDF文件合并步骤 | 能力型 | 纯技术操作 |
