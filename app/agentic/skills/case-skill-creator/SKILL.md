# 文章转技能 (Article to Skill)

将文章内容理解、重组为结构化的 Claude 标准技能。

---

## 工作流程

```
Step 1: 获取内容 → Step 2: 分析类型 → Step 3: 创建技能 → Step 4: 事实校验 → Step 5: 输出
```

---

## Step 1: 获取输入内容

**文本输入**：直接使用用户提供的文本。

**链接输入**：使用 `read_url_content` 工具获取页面内容。

**图文内容**：提取图片中的关键信息，结合文字理解。

---

## Step 2: 分析内容类型

根据文章特征**自动判断**：

| 主要特征 | 类型 | 技能结构 |
|---------|------|---------|
| 法条、定义、规则、标准 | 知识型 | 概念体系 + 规则库 + 速查表 |
| 操作步骤、流程、SOP | 能力型 | 分步SOP + 决策点 + 检查清单 |
| 两者兼有且强耦合 | 混合型 | 知识部分 + 操作部分分层 |

**快速判断**：
- "是什么"、"定义"、"包括" → 知识型
- "怎么做"、"步骤"、"流程" → 能力型
- 操作依赖概念判断 → 混合型

---

## Step 3: 创建技能

### 3.1 目录结构

```
skill-name/
├── SKILL.md              # 主技能文件
└── references/           # 辅助资料
    ├── source.md         # 原文备份（可选）
    └── [图片等资源]
```

### 3.2 SKILL.md 编写

**理解并重组**原文内容，按选定类型的结构编写：

#### 知识型结构
```markdown
# [主题] 知识库

## 概念体系
[从原文提取并组织概念定义]

## 规则库
[从原文提取规则，按类别组织]

## 速查表
[关键信息表格化]
```

#### 能力型结构
```markdown
# [功能] SOP

## 适用场景
[说明何时触发]

## Step 1: [阶段名]
[从原文提取步骤]

## Step 2: [阶段名]
[继续步骤...]

## 决策要点
[条件判断表格]
```

#### 混合型结构
```markdown
# [主题]

## 第一部分：知识基础
[概念和规则]

## 第二部分：操作流程
[SOP，引用第一部分概念]
```

### 3.3 资源处理

- 图片：保存到 `references/` 并在 SKILL.md 中引用
- 长法条：可保留到 `references/laws.md` 供深度查阅

---

## Step 4: 事实校验（关键步骤）

> [!IMPORTANT]
> 技能创建完成后，**必须执行此步骤**进行事实核对。

### 4.1 校验清单

对照原文逐项检查：

| 校验项 | 检查内容 | 严重等级 |
|-------|---------|---------|
| 法条编号 | 条、款、项编号是否一致 | 🔴 严重 |
| 数字数据 | 金额、比例、期限是否准确 | 🔴 严重 |
| 关键术语 | 专业术语是否正确使用 | 🔴 严重 |
| 因果关系 | 逻辑关系是否正确表述 | 🟠 中等 |
| 限定条件 | 例外情况是否遗漏 | 🟠 中等 |
| 流程顺序 | 步骤先后是否正确 | 🟡 一般 |

### 4.2 校验方法

```
1. 逐段对比：将技能内容与原文逐段对照
2. 关键词提取：检查法条号、数字、专业术语
3. 逻辑验证：确认因果/条件关系未被扭曲
4. 完整性检查：确认关键信息未遗漏
```

### 4.3 发现问题时

| 问题类型 | 处理方式 |
|---------|---------|
| 法条编号错误 | **立即修正**，以原文为准 |
| 数字不符 | **立即修正**，以原文为准 |
| 逻辑表述不当 | 重新表述，保持原意 |
| 遗漏重要内容 | 补充完整 |
| 过度简化 | 评估是否影响准确性，必要时补充 |

---

## Step 5: 输出

### 5.1 命名规范

- Skill 名称：`kebab-case` 格式
- 文件夹命名应能体现技能主题

### 5.2 Frontmatter 要求

```yaml
---
name: skill-name
description: "简要描述。当[触发条件]时使用。"
---
```

### 5.3 最终检查

- [ ] 目录结构符合 Claude 标准
- [ ] SKILL.md 包含完整的 frontmatter
- [ ] 内容已通过事实校验
- [ ] 图片等资源正确保存和引用
