# 原子提取指南

## 原子类型识别

### Case Atom（案例原子）→ A类

**识别条件**:
- 文章出现"某类纠纷的典型情境"
- 有法院认定要点/结果/败因
- 哪怕不是完整故事，也能补成A类

**关键词**: 案例、判决、败诉、胜诉、裁判要点、法院认定

---

### Misconception Atom（误解原子）→ B类

**识别条件**:
- 文章出现"常见误区/很多人以为/其实不是/注意"
- 能提取用户常见问法和错误理解

**关键词**: 误区、常见问题、很多人以为、其实、注意、实际上

---

### Gap Atom（缺口原子）→ C类

**识别条件**:
- 文章出现"关键证据/必须满足/否则不支持/举证责任/缺了就难"
- 能落成"最低门槛一句话"

**关键词**: 关键证据、必须、否则、举证责任、要件、缺失

---

## 适配评分标准

### A类适配条件（得分项）

- ✓ 有具体情境（交易/欠款/拖欠/争议点）
- ✓ 有败因/争点/裁判要点
- ✓ 能提炼一句"劝退门槛"

### B类适配条件

- ✓ 文中出现"误区/常见问题/很多人以为/其实"
- ✓ 能抽出一句"纠偏结论"

### C类适配条件

- ✓ 明确写到"证据/要件/举证责任/必须/否则"
- ✓ 能落成"最低门槛一句话"

---

## 优先级规则

> **三类都能做时，优先级: C > B > A**

原因：缺口/门槛最像"尺子卡片"，最短、最可量产、最不依赖剧情细节。

---

## 不适用处理

当文章不适配任何原子类型时：

| 输出 | 说明 |
|-----|------|
| `DROP` | 丢弃，不产出脚本 |
| `RESEARCH` | 仅进入选题库，不产出脚本 |

**原因标签**（必须标注）:

| 标签 | 含义 |
|-----|------|
| `NO_RULE` | 没有可提炼规则/门槛 |
| `NO_GAP` | 没有证据缺口 |
| `NO_CASE` | 没有情境与争点 |
| `TOO_TUTORIAL` | 过于教程化，不符合内容策略 |
