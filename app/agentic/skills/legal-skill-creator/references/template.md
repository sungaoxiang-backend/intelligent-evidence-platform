# 法律知识→技能 转化模板

## 模板结构

每个法律技能应包含以下三个层次：

### 1. 触发层（Frontmatter）

```yaml
---
name: [技能名-使用连字符]
description: |
  当用户说"[指令模式1]"或"[指令模式2]"时，使用此技能执行：
  1. [任务步骤1]
  2. [任务步骤2]
  3. [任务步骤3]
---
```

**关键原则**：
- description 必须是**可执行的动作列表**
- 不是"帮助用户理解XX"，而是"执行XX"
- 指令模式要具体：不是"关于证据的问题"而是"帮我审核这份证据"

### 2. 任务层（SKILL.md Body）

```
# [技能名]

## 快速开始

当用户说"[核心指令]"时：

### 步骤1：[操作名称]
**输入**：用户提供的[具体材料]
**操作**：
1. [具体行动1]
2. [具体行动2]
**输出**：[产出物]

### 步骤2：[操作名称]
...

## 详细指南

### 场景A：[具体场景]
当[条件满足]时：
- 执行[操作]
- 参考[知识条目]

### 场景B：[具体场景]
...
```

### 3. 知识层（references/）

将文章中的详细知识放入 references，按需加载：
- `references/knowledge.md` - 核心法律知识
- `references/examples.md` - 案例和示例
- `references/checklist.md` - 检查清单

---

## 转化检查表

### Frontmatter 检查

- [ ] name 使用 kebab-case（连字符）
- [ ] description 包含具体任务步骤
- [ ] 指令模式是用户会说的原话
- [ ] 任务步骤是动词开头的动作

### 任务层检查

- [ ] 有"快速开始"部分
- [ ] 每个步骤有输入/操作/输出
- [ ] 用户能按步骤执行
- [ ] 不是知识讲解，而是操作指引

### 知识层检查

- [ ] 详细知识在 references，不在 SKILL.md
- [ ] SKILL.md 只保留操作必需的信息
- [ ] 引用清晰：需要时查阅 references/xxx.md

---

## 示例：证据审核技能

### Frontmatter（正确）
```yaml
---
name: evidence-authenticator
description: |
  当用户说"帮我审核这份证据"或"看看这个能当证据吗"时，使用此技能执行：
  1. 要求用户提供证据材料
  2. 按三性（真实性、合法性、关联性）逐项检查
  3. 输出审核结论和改进建议
---
```

### Body（正确）
```
# 证据审核

## 快速开始

当用户说"帮我审核证据"时：

### Step 1: 收集证据信息
请用户提供：
- 证据类型（书证/物证/电子数据等）
- 证据原件还是复印件
- 证据来源和形成过程

### Step 2: 三性检查
依次检查：
1. **真实性**：载体是否真实？内容是否可靠？
2. **合法性**：来源合法？形式合法？
3. **关联性**：与待证事实有关联？

### Step 3: 输出结论
根据检查结果：
- 证据是否具有证据能力
- 证据证明力强弱
- 补充建议
```
