# 7维风格分析框架 (7-Dimension Style Analysis Framework)

本文档提供完整的风格分析框架，用于分析文章/脚本的口播风格。

---

## 维度1：人设与关系 (Persona & Audience)

### 分析要点
- **讲述者像谁？** 律师/顾问/老师/朋友/警示者？
- **对观众的称呼与站位？** "你"/"大家"/"老板们"/"同学们"？
- **关系定位？** "我带你判断" vs "我给你尺子" vs "我替你拆雷"

### 特征总结示例

**专业顾问型：**
- 用"律师告诉你""我处理过上百个案例"
- 站位是"我有经验，我教你"
- 给人专业、权威的感觉

**警示者型：**
- 用"别白忙""别踩坑""听我一句劝"
- 站位是"我替你踩过坑，我提醒你"
- 给人紧迫感、警示感

**朋友型：**
- 用"我跟你说""我跟你说实话"
- 站位是"我也是这么过来的"
- 给人亲切、信任感

### 可操作规则
1. 明确人设：先选一个身份（律师/顾问/警示者/朋友）
2. 统一称呼：全文统一用"你"或"大家"
3. 一致站位：不要一会是专家，一会是朋友
4. 开头立人设：第一句就要让观众知道你是谁

### 典型句式模板
- "律师告诉你..."
- "我处理过XX个案例，告诉你一个规律..."
- "别白忙，听我一句劝..."
- "我跟你说实话..."

## 维度2：语气与情绪曲线 (Tone & Emotional Arc)

### 分析要点
- **开头语气？** 强势压迫/冷静理性/共情安抚/悬念设问？
- **中段推进？** 递进/对比/反差/案例？
- **结尾语气？** 收敛/鼓励/警示/号召行动？

### 特征总结示例

**警示型情绪曲线：**
- 开头：强警示，用"最怕""千万别"制造紧张感
- 中段：用"你以为...实际..."制造反差，强化紧张
- 结尾：给出解决方案，缓解紧张，引导行动

**理性分析型情绪曲线：**
- 开头：冷静提问，引发思考
- 中段：用数据和案例，逻辑推理
- 结尾：给出判断标准，让用户自己决定

**共情型情绪曲线：**
- 开头：用"我也遇到过"拉近距离
- 中段：讲用户痛点，表示理解
- 结尾：给出帮助，温暖收尾

### 可操作规则
1. 开头3秒定调子：第一句就确定情绪基调
2. 中段保持情绪：不要一会儿紧张一会儿松弛
3. 结尾要有行动：情绪收束到CTA（Call to Action）
4. 反差制造张力：用"你以为...实际..."制造情绪起伏

### 典型句式模板
- **开头：** "最怕..."/"千万别..."/"你可能不知道..."
- **中段：** "你以为...实际..."/"很多人觉得...但..."/"问题出在哪？"
- **结尾：** "别白忙..."/"先自查..."/"评论区扣..."

## 维度3：口语节奏与句子结构 (Spoken Rhythm & Syntax)

### 分析要点
- **短句/长句比例？** 口播多用短句（5-15字）
- **停顿点？** 逗号/顿号/换行怎么用？
- **修辞手法？** 反问/递进/对比/排比/三段式/重复强调？

### 特征总结示例

**快节奏型：**
- 80%是短句（5-10字）
- 多用逗号，一逗到底
- 用"第一...第二...第三..."推进
- 适合60秒短视频

**反差型：**
- 用"你以为...实际..."制造反差
- 用"很多人觉得...但..."反转
- 用"问题不在...而在..."揭示真相

**三段式型：**
- 用"是什么...为什么...怎么办..."结构
- 用"定义→后果→解决"三步走
- 节奏清晰，观众易跟随

### 可操作规则
1. 短句为主：平均句长10-15字
2. 逗号停顿：用逗号控制呼吸节奏
3. 换行分段：一个观点一段，不要一段3-4个观点
4. 修辞制造节奏：用反问、对比、三段式增强节奏感

### 典型句式模板
- **短句：** "这钱拖了8个月。30万要不回来。问题出在哪？"
- **反问：** "你以为有合同就稳了？错！"
- **对比：** "很多人觉得...但实际..."
- **三段式：** "第一，...第二，...第三，..."
- **递进：** "不仅是...更是..."/"不仅是...而且...甚至还..."

## 维度4：用词与口头禅 (Vocabulary & Catchphrases)

### 分析要点
- **大白话 vs 专业术语？** 口播多用大白话
- **术语解释方式？** 用类比/比喻
- **高频词/口头禅？** 固定搭配、常用句式

### 特征总结示例

**大白话型：**
- "空壳公司"代替"无资产运营主体"
- "白忙"代替"无效投入"
- "翻脸不认账"代替"违约否认"
- 术语占比 < 10%

**专业+解释型：**
- 用"什么叫...？就是..."解释术语
- 用类比："执行缺口就像...，你赢了官司也拿不到钱"
- 术语占比 20-30%，但都解释了

### 可操作规则
1. 优先大白话：能用大白话就不用术语
2. 术语必解释：用"什么叫...？就是..."
3. 用类比/比喻：让观众一听就懂
4. 固定搭配：形成个人特色的口头禅

### 典型句式模板
- **解释术语：** "什么叫主体缺口？就是..."
- **类比：** "就像...一样"/"好比..."
- **口头禅：** "记住：..."/"千万别..."/"我跟你说..."

## 维度5：脚本结构母版 (Script Blueprint)

### 分析要点
- **结构顺序？** Hook → 背景 → 误判 → 真相 → 方案 → CTA
- **段落长度？** 每段句数/秒感
- **段落过渡？** 过渡句怎么写？

### 标准结构模板

**60-90秒脚本结构：**
```
1. 开头钩子（Hook） - 3-5秒，1-2句
2. 背景/案情 - 10-15秒，2-3句
3. 误判点 - 5-10秒，1-2句
4. 真相/真卡点 - 10-15秒，2-3句
5. 规则/结论 - 5-10秒，1-2句
6. CTA（行动号召） - 5-10秒，1-2句
```

### 可操作规则
1. 开头抓眼球：前3秒必须有悬念/痛点/数字
2. 段落清晰：一个观点一段，不要混在一起
3. 过渡自然：用"问题出在哪？""为什么？"等过渡
4. 结尾CTA：最后一定要有明确的行动指令

### 典型句式模板
- **开头钩子：** "XX拖了X个月，XX万要不回来，为什么？"
- **过渡句：** "问题出在哪？"/"为什么？"/"很多人觉得...但..."
- **CTA：** "私信【评估】，领《回款判断尺子卡》"

## 维度6：论证与信息密度 (Reasoning Style)

### 分析要点
- **结论先行 vs 故事先行？** "先结论后理由"或"先故事后抽象"
- **证据/例子嵌？** 用案例/数据/对比
- **反差制造？** 用"如果...那就...""你以为...其实..."

### 特征总结示例

**结论先行型：**
- 先说"回款最怕主体缺口"
- 再解释"什么叫主体缺口"
- 最后说"怎么判断"
- 适合干货型内容

**故事先行型：**
- 先讲案例"供应商货款拖了8个月"
- 再揭示"实际输在主体"
- 最后总结"记住：主体是空壳，签了也是白签"
- 适合案例型内容

**反差型：**
- 用"你以为...实际..."制造反差
- 用"很多人觉得...但..."反转认知
- 适合纠偏/辟谣型内容

### 可操作规则
1. 选一个论证逻辑：结论先行/故事先行/反差型
2. 全文统一：不要一会故事一会结论
3. 证据要具体：用数字/案例/对比
4. 反差要明显：用"你以为...实际..."强化记忆点

### 典型句式模板
- **结论先行：** "回款最怕..."/"XX的关键是..."
- **故事先行：** "XX拖了X个月，XX万要不回来"
- **反差：** "你以为...实际..."/"很多人觉得...但..."
- **假设：** "如果...就..."/"不...就..."

## 维度7：互动与指令性 (CTA & Guidance)

### 分析要点
- **CTA类型？** 评论/私信/关注/发材料/自查
- **CTA语气？** 命令式/建议式/引导式
- **免责声明？** 是否有"边界提醒"

### 特征总结示例

**引导式CTA：**
- "私信【评估】，领《回款判断尺子卡》"
- "评论区扣'尺子'，发给你自查工具"
- 语气是邀请，不是命令

**命令式CTA：**
- "赶紧自查！"
- "别等了，现在就要！"
- 语气强烈，制造紧迫感

**建议式CTA：**
- "建议先自查，再决定要不要追"
- "你可以对照看看"
- 语气温和，给用户选择权

### 可操作规则
1. CTA要具体：不要"关注我"，要说"私信【评估】"
2. CTA要有价值：明确告诉用户能得到什么
3. CTA要统一：每次都用同样的CTA，形成记忆
4. 适当免责：敏感内容加"具体咨询专业律师"

### 典型句式模板
- **引导式：** "私信【评估】，领《回款判断尺子卡》"
- **命令式：** "评论区扣'尺子'，马上发给你"
- **建议式：** "建议先自查，再决定要不要追"
- **免责：** "具体问题建议咨询专业律师"

## 风格分析检查清单

使用这个框架分析文章/脚本时，按以下顺序检查：

- [ ] **人设与关系**：讲述者是谁？对观众的称呼和站位？
- [ ] **语气与情绪曲线**：开头/中段/结尾的语气和情绪推进？
- [ ] **口语节奏与句子结构**：短句/长句比例？修辞手法？
- [ ] **用词与口头禅**：大白话还是术语？有哪些口头禅？
- [ ] **脚本结构母版**：结构顺序？段落长度？过渡句？
- [ ] **论证与信息密度**：结论先行还是故事先行？如何制造反差？
- [ ] **互动与指令性**：CTA类型？语气？是否有免责声明？

完成这7个维度的分析后，就能生成完整的《风格画像报告》。
