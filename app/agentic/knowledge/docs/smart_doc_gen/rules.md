# 智能文档生成规则与示例

## 1. 基本原则
- **准确性 (Accuracy)**: 填充内容必须严格依据案件数据 (Case Data)。严禁编造信息。
- **仅填充 (Fill Only)**: 严禁删除或修改模板原有的任何内容（包括不适用的字段）。仅在空白处填写内容，或勾选复选框。
- **格式规范 (Formatting)**: 除非另有说明，日期应使用 `YYYY年MM月DD日` 格式。金额应包含单位（如 `元`）。

## 2. 当事人角色分析规则 (Mental Model)

### 原告分析
- **如果原告是自然人 (`person`):**
    - **忽略**: "法定代表人"、"统一社会信用代码"、"住所地/注册地" 等企业专用字段。**不要填写，也不要删除，保持原样。**
    - **填写**: "姓名"、"身份证号"、"住址/经常居住地"、"联系方式"。
- **如果原告是公司 (`company`):**
    - **填写**: "名称/企业名称"、"法定代表人"、"统一社会信用代码"、"住所地"。

### 被告分析
- **如果被告是自然人 (`person`):**
    - **忽略**: "法定代表人"、"统一社会信用代码"、"住所地/注册地"。**不要填写，也不要删除，保持原样。**
    - **填写**: "姓名"、"身份证号" (如有)、"住址"、"联系方式"。
- **如果被告是公司 (`company`):**
    - **填写**: "名称/企业名称"、"法定代表人"、"统一社会信用代码"、"住所地"。

## 3. 学习示例 (Few-Shot Examples)

### 示例 A: 自然人 vs 公司
**场景**: 原告是张三 (自然人)，被告是李四有限公司 (公司)。
**模板区块**: 
```
原告：[Name]
法定代表人：[LegalRep]
被告：[DefName]
法定代表人：[DefLegalRep]
```
**正确填充**:
- `Name` -> "张三"
- `LegalRep` -> (省略/不放入映射) (原因: 原告是自然人，留空)
- `DefName` -> "李四有限公司"
- `DefLegalRep` -> "王五" (原因: 被告是公司，正常填写)

### 示例 B: 公司 vs 自然人
**场景**: 原告是甲公司 (公司)，被告是乙某 (自然人)。
**模板区块**:
```
原告：[Name]
身份证号：[ID]
法定代表人：[LegalRep]
被告：[DefName]
法定代表人：[DefLegalRep]
```
**正确填充**:
- `Name` -> "甲有限公司"
- `ID` -> (省略/不放入映射) (原因: 原告是公司，通常不填身份证号)
- `LegalRep` -> "赵六"
- `DefName` -> "乙某"
- `DefLegalRep` -> (省略/不放入映射) (原因: 被告是自然人，留空)
