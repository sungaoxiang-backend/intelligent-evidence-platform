# Change: 添加文书管理页面

## Why

当前系统存在旧的模板管理功能（`document-templates`）和文书生成功能（`document-generation`），但这些模块存在问题。用户需要一个全新的、完全独立的"文书管理"功能模块，用于：
- 检索和预览已创建的文书
- 直接创建新文书（不依赖模板）
- 编辑已存在的文书
- 保证预览和编辑时内容和表格格式的统一性

**重要原则**：本模块完全独立实现，不与现有模板管理、文书生成模块产生任何耦合，为后续逐步替换旧模块做准备。

## What Changes

- **新增文书管理页面**：在顶部导航添加"文书管理"入口，创建 `/documents` 路由页面
- **文书列表和检索**：支持检索文书列表和预览具体文书，显示文书基本信息
- **新增文书功能**：提供"新增文书"按钮，进入基于 Tiptap 的富文本编辑器创建新文书
- **编辑文书功能**：在预览模式下提供"编辑"按钮，进入富文本编辑模式编辑已存在的文书
- **预览和编辑模式切换**：支持在预览和编辑模式之间切换，点击"取消"或"保存"返回预览
- **格式一致性保证**：确保预览和编辑时内容和表格格式的统一性
- **后端 API 支持**：创建文书管理的后端 API，包括 CRUD 操作

## Impact

- **Affected specs**: 
  - `document-management` - 新增文书管理能力规范
- **Affected code**: 
  - `frontend/app/documents/` - 新增文书管理页面
  - `frontend/components/top-navigation.tsx` - 添加"文书管理"导航项
  - `frontend/components/documents/` - 新增文书相关组件（列表、预览、编辑器）
  - `app/documents_management/` - 新增后端模块（模型、服务、路由）
  - `app/api/v1.py` - 注册新的路由
