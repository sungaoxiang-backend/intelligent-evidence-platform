# Change: 优化文书管理模块交互体验

## Why

当前文书管理模块存在以下用户体验问题：

1. **编辑模式按钮混乱**：编辑模式下意外显示了"导出 PDF"按钮，但用户期望编辑模式下只显示"保存"和"取消"按钮，导出功能应该在预览模式下使用。
2. **保存时缺少命名功能**：保存新文书时，系统自动使用"新文书"作为默认名称，用户无法自定义命名。对于已存在的文书，更新时也无法修改名称。
3. **列表操作不便**：侧栏的文书列表只支持点击查看，缺少快速操作按钮（编辑、删除），用户需要先点击进入预览，再点击编辑，操作路径较长。

这些问题的存在降低了用户的工作效率，需要优化交互流程。

## What Changes

- **优化编辑模式按钮显示**：编辑模式下移除"导出 PDF"按钮，只显示"保存"和"取消"按钮。导出功能保留在预览模式下。
- **添加保存时命名功能**：保存新文书或更新文书时，弹出对话框让用户输入或修改文书名称。如果文书已有名称，则自动填充到输入框中。
- **增强列表操作功能**：在侧栏文书列表的每个文书卡片上添加"编辑"和"删除"按钮，支持快速操作。点击"编辑"直接进入编辑模式，点击"删除"弹出确认对话框后删除。

## Impact

- **Affected specs**: 
  - `document-management` - 修改文书管理能力规范，优化交互流程
- **Affected code**: 
  - `frontend/app/documents/page.tsx` - 修改保存逻辑，添加命名对话框
  - `frontend/components/documents/document-editor.tsx` - 移除编辑模式下的导出按钮
  - `frontend/components/documents/document-preview.tsx` - 确保预览模式下有导出按钮
  - `frontend/components/documents/document-list.tsx` - 添加编辑和删除按钮
  - `frontend/lib/documents-api.ts` - 确保 API 支持更新名称（已支持）
